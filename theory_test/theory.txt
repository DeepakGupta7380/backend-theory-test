Q:1. What is Node.js? Explain how it works internally with the event loop.

Ans: Node.js is a Javascript runtime built on the V8 engine that uses a single-threaded event loop with
non blocking I/O to efficiently handle many concurrent operations by delegating asynchronous work and executing
callbacks when the main thread is free.

Q:2. Difference between synchronous and asynchronous programming in Node.js. Give examples.

Ans: Synchronous programming executes tasks sequentially and blocks the execution untill each task completes, whereas
asynchronous programming starts a task and allowed the program to continue executing other code while waiting for the
task to finish. In Node.js, asynchronous programming is preferred because it uses the Event Loop and non-blocking I/on
to handle many concurrent operations efficiently.

Q:3. 3. Create a Node.js server using the built-in http module.

Ans: the built-in module in Node.js is used to create HTTP servers without any external library. you create a server using
https.createServer(), handle the incoming request (req) and response (res) objects, and start listening for connections 
with server.listen(port).

Q:4. What is npm? Explain the difference between dependencies and devDependencies.

Ans: npm (Node Package Manager) is the default package manager for Node.js that helps install, manage, and share Javascript
packages. dependencies are packages required for the application to run in production, while devDependencies are packages
needed only during development, testing or building the project.


Q:5. Explain the purpose of package.json and package-lock.json.

Ans: . package.json is the projects manifest file that stores metadata, scripts, and the list of required dependencies.

. package-lock.json is automatically generated by npm and records the exact versions of all installed packages and 
reproducible installations across different machines.


Q:6. 6. What is Express.js? Why do we use Express instead of the Node.js HTTP module?

Ans: Express.js is a lightweight web framework built on top of Node.js that simplifies server-side development. It provides
features like routing, middleware, and request handling. We use Express instead of the raw Node.js http module because it
reduces boilerplate code, improves readability, and makes it easier to build scalable APIs and web applications.

Q:8. 8. Explain middleware in Express.js. Create a custom middleware for logging request details.

Ans: Middleware in Express.js is a function that executes during the request-response cycle and has access to request, 
response, and next. It is used for tasks like logging, authentication, validation, and request processing.
A custom middleware can be created to log request details by capturing method, URL and timestamp, then calling next() to pass
control to the next handler.


Q:9. What is the difference between req.params, req.query, and req.body?

Ans: In Express.js, req.params is used to access route parameters in the URL path, req.query is used to access query string
parameters for filtering or searching, and req.body is used to access data sent in the request body , usually in POST or PUT requests.


Q:10. Create an authentication middleware that checks whether a user has a valid token.

Ans: An authentication middleware in Express.js is a function that checks whether a request contains a valid token before
allowing access to protected routes. It reads the token from request headers, validates it, and either calls next()
to proceed or returns an unauthorized error if the token is missing or invalid.


Q:11. Explain the MVC architecture in backend development. How will you structure a Node + Express project?


Ans: MVC (Model-View-Controller) is a design pattern used to structure backend applications. The Model handles database 
logic, the Controller handles business logic and request processing, and the client. In a Node.js + Express project, 
we typically organize code into seprate folders for models, controllers, routes, and middleware to improve scalability,
maintainability, and readability.

Q:13. What is MongoDB? Explain the difference between SQL databases and MongoDB.


Ans: MongoDB is a NoSQL document-oriented database that stores data in flexible JSON-like documents. Unlike SQL database
which use structure tables with fixed schemas, MOngoDB uses collections and documents with dynamic schemas. SQL is 
best for structure relational data and complex jions, while MongoDB is better for unstructured or semi-structured data and 
scalable modern application like MERN stack projects.

Q:15. Explain the difference between find(), findOne(), and findById() in MongoDB.

Ans: In MongoDB, find() returns multiple documents as an array based on a filter, findOne() returns 
the first matching single document, and findById() is a shortcut to find a document using its unique _id 
field. find() 
is used for listing data, while findOne() and findById() are used when a single record is needed.
